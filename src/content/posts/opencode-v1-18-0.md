[`opencode`](https://github.com/anomalyco/opencode) выпустил минорный релиз [`v1.18.0`](https://github.com/anomalyco/opencode/releases/tag/v1.18.0). В этом выпуске завершена миграция Desktop на новый интерфейс v2, а приложение стало заметно устойчивее в длинных сессиях: Home быстрее загружается, история таймлайна подгружается крупными страницами и корректно восстанавливается после reconnect.

Источник статьи — [GitHub Release `v1.18.0`](https://github.com/anomalyco/opencode/releases/tag/v1.18.0), [compare `v1.17.20...v1.18.0`](https://github.com/anomalyco/opencode/compare/v1.17.20...v1.18.0) и изменения в исходном коде релиза. Ниже перечислены не только пункты changelog, но и конкретное поведение, добавленное в связанных коммитах.

## Desktop v2 становится основным интерфейсом

Финальный коммит миграции [#36912](https://github.com/anomalyco/opencode/pull/36912) завершает перенос Desktop на v2. Новые пользователи сразу получают новый интерфейс без возврата к v1. Для существующих профилей приложение учитывает прежние настройки, серверы, проекты, сессии и WSL-серверы: они переходят на v2, но на время переходного периода могут вернуться к старому layout.

Переключатель добавлен в общие настройки отдельным PR [#36667](https://github.com/anomalyco/opencode/pull/36667). Это именно временный migration control, а не постоянная настройка продукта: после наступления даты sunset пользователи, оставшиеся на старом интерфейсе, автоматически переводятся на v2, а переключатель заменяется закрываемым уведомлением. Классификация профиля сохраняется, поэтому при каждом запуске не требуется заново исследовать пользовательские данные. До определения layout стартовый экран удерживается на splash, чтобы не было вспышки между v1 и v2.

В релиз также вошли связанные исправления Desktop v2:

- файловые представления используют базовый фон v2 вместо фона старого layout — [коммит `bfddf057`](https://github.com/anomalyco/opencode/commit/bfddf057);
- onboarding первого запуска корректно пропускается или показывается в зависимости от состояния миграции — [коммит `dffecb64`](https://github.com/anomalyco/opencode/commit/dffecb64);
- после обновления обрабатывается переход пользователя на новую структуру интерфейса — [коммит `4a181c35`](https://github.com/anomalyco/opencode/commit/4a181c3574875bcf780f17e6275805cadbb4eb57).

## Home: холодная загрузка и индекс сессий

Коммит [#36214](https://github.com/anomalyco/opencode/commit/449c64928b0dc5ddb47f0e85d9b65323f79a267e) меняет загрузку Home: пассивное создание store для каталога больше не включает все directory-запросы автоматически. Запросы providers, LSP, references и path активируются только при реальном обращении к каталогу, а не во время предварительного отображения Home.

Для списка сессий добавлен отдельный v2-индекс (`home-session-index.ts`). Он загружает данные глобальным запросом страницами по 5 000 записей (`limit: 5000`, `order: "desc"`), продолжая запрос по `cursor`, пока сервер не вернёт последнюю страницу. Затем индекс применяет более новые события `session.created`, `session.updated` и `session.deleted`, чтобы результат не потерял изменения, пришедшие во время загрузки.

Практический эффект подтверждён названием и тестами коммита: Home cold load ускорен в 78 раз на соответствующем сценарии, а тяжёлые запросы для отдельных директорий не запускаются до фактического открытия проекта. При reconnect или перемещении сессии индекс обновляется, а архивированная сессия, которой нет в пассивном store, больше не добавляется обратно из события.

## Таймлайн: меньше backfill и корректное восстановление после reconnect

Для таймлайна изменена стратегия подгрузки истории. Коммит [#36172](https://github.com/anomalyco/opencode/commit/729bd438) предварительно загружает больше сообщений за один запрос, поэтому прокрутка к границе истории реже запускает повторный backfill. При этом тесты проверяют сохранение видимого сообщения на месте, пока сверху добавляются старые записи.

После восстановления маршрута virtualizer может увидеть DOM-элемент заново подключённым, но получить устаревшее смещение. Новый reconnect-aware observer в [коммите `7304bb27`](https://github.com/anomalyco/opencode/commit/7304bb27) наблюдает за удалением и повторным добавлением scroll element, сверяет фактический `scrollTop` с внутренним offset TanStack Virtual и отправляет расхождение обратно в virtualizer. Проверка продолжается ещё несколько animation frames после `isScrollingResetDelay`, поэтому старый callback не может перезаписать актуальную позицию.

Для горизонтального режима учитывается направление RTL: фактический `scrollLeft` передаётся со знаком, соответствующим настройкам virtualizer. Для вертикального таймлайна это означает сохранение нижнего якоря после reconnect вместо внезапного прыжка к пустому участку.

## Permission auto-accept разделён по серверам

Раньше состояние автоматического принятия permissions могло быть общим для разных подключённых серверов. В [коммите `e7e7a976`](https://github.com/anomalyco/opencode/commit/e7e7a976) состояние привязано к server key. Поэтому включение auto-accept для одного local или remote server не меняет поведение другого.

Это особенно важно для remote sessions: [коммит `cdca203b`](https://github.com/anomalyco/opencode/commit/cdca203b) исправляет передачу этого состояния в удалённую сессию. Теперь permission request, пришедший через remote connection, использует настройки именно своего сервера и действительно может быть автоматически принят.

Сценарий можно представить так:

```text
server A: auto-accept = on
server B: auto-accept = off

permission request от A → принимается автоматически
permission request от B → остаётся на ручное подтверждение
```

Это не новая пользовательская команда, а исправление области действия уже существующей настройки: конфигурация одного сервера больше не протекает в другой.

## Проектный picker и фокус терминала

В связанных исправлениях project picker теперь ждёт, пока anchor-элемент подключён и получил позицию, вместо попытки вычислить координаты по ещё не готовому DOM. Отдельный guard предотвращает crash при позиционировании picker — [коммиты `5f7091ab`](https://github.com/anomalyco/opencode/commit/5f7091ab) и [`f58b8cb6`](https://github.com/anomalyco/opencode/commit/f58b8cb6).

Для терминалов релиз исправляет сразу два похожих сценария:

- переименование вкладки сохраняет фокус input во время редактирования — [коммит `59fe3509`](https://github.com/anomalyco/opencode/commit/59fe350972fde0aa96f3f13ab243befb23d48a23);
- подключение Ghostty и монтирование terminal не крадут фокус у composer — [коммит `a1ed9927`](https://github.com/anomalyco/opencode/commit/a1ed99278a68d0d54b7fb40985fceece8b51c0a2).

Внутри terminal session появился явный focus request с идентификатором запроса и терминала. Новый `Terminal` получает `onAutoFocus`, а восстановление focus запускается только для терминала, который действительно запросил фокус. Если пользователь успел переключиться на composer, завершившееся монтирование не возвращает focus обратно в terminal. Для кнопки нового терминала, наоборот, можно явно передать `terminal.new({ focus: true })`.

```ts
terminal.new({ focus: true });
```

Этот вызов подтверждён изменениями в `context/terminal.tsx` и `terminal-panel-v2.tsx`; обычное монтирование уже открытого terminal не означает автоматический focus.

## Мелкие исправления, заметные в ежедневной работе

В релиз вошли ещё несколько исправлений состояния UI:

- caret composer сохраняется после обновления request — [коммит `a625d35f`](https://github.com/anomalyco/opencode/commit/a625d35f);
- нижний якорь таймлайна сохраняется надёжнее при росте активного содержимого — [коммит `35c88c3f`](https://github.com/anomalyco/opencode/commit/35c88c3f);
- severity status indicator теперь различается понятнее — [коммит `b5e09024`](https://github.com/anomalyco/opencode/commit/b5e09024);
- terminal tab больше не получает неожиданный focus только потому, что компонент смонтировался;
- для timeline добавлены regression и stability-тесты, проверяющие prepend истории, reconnect, virtualization, resizing и сохранение focus.

## Итог

`opencode 1.18.0` — прежде всего релиз завершения Desktop v2 и стабилизации нового интерфейса. Существующие пользователи получают контролируемый переход с временным переключателем, а новые — сразу v2 без устаревшего режима. Для повседневной работы важнее всего ускоренный Home, крупнее порции истории таймлайна, восстановление позиции после reconnect, изоляция permission auto-accept по серверам и предсказуемый focus composer/terminal.
