Zed выпустил minor-релиз [`v1.10.0`](https://github.com/zed-industries/zed/releases/tag/v1.10.0). Это не pointer-only release: основной источник — GitHub Release и связанные PR. В релизе есть три изменения, которые напрямую меняют повседневную настройку редактора: `llama.cpp` стал отдельным language model provider, blame текущей строки можно перенести из inline-текста в status bar, а глобальный `format_on_save` теперь выключен по умолчанию для языков без официального formatter'а.

Источник для обзора — GitHub Release [`zed-industries/zed@v1.10.0`](https://github.com/zed-industries/zed/releases/tag/v1.10.0) и PR [`#59964`](https://github.com/zed-industries/zed/pull/59964), [`#59696`](https://github.com/zed-industries/zed/pull/59696), [`#59710`](https://github.com/zed-industries/zed/pull/59710), [`#59860`](https://github.com/zed-industries/zed/pull/59860), [`#56867`](https://github.com/zed-industries/zed/pull/56867), [`#59310`](https://github.com/zed-industries/zed/pull/59310), [`#59806`](https://github.com/zed-industries/zed/pull/59806), [`#59958`](https://github.com/zed-industries/zed/pull/59958), [`#55489`](https://github.com/zed-industries/zed/pull/55489), [`#59567`](https://github.com/zed-industries/zed/pull/59567), [`#59840`](https://github.com/zed-industries/zed/pull/59840) и [`#59849`](https://github.com/zed-industries/zed/pull/59849).

## `llama.cpp` больше не надо маскировать под OpenAI-compatible provider

PR [`#59964`](https://github.com/zed-industries/zed/pull/59964) добавляет в Zed отдельный provider `llama_cpp`. До этого пользователи `llama serve` могли подключаться через generic OpenAI-compatible provider, но теряли главное удобство router mode: `llama.cpp` умеет отдавать список моделей из локального cache и загружать их по требованию, а generic provider требовал вручную описывать каждую модель и её capabilities.

В diff появился новый crate `crates/llama_cpp`, provider `crates/language_models/src/provider/llama_cpp.rs`, иконка `assets/icons/ai_llama_cpp.svg`, документация в `docs/src/ai/use-a-local-model.md` и Settings Editor schema в `crates/settings_content/src/language_model.rs`. Новый клиент работает с OpenAI-compatible chat endpoint'ом `/v1/chat/completions`, но отдельно поддерживает discovery через `/v1/models` и `/props`.

Практически это означает, что локальный `llama.cpp` можно описать как отдельного провайдера, а не как набор custom OpenAI-compatible моделей:

```json
{
  "language_models": {
    "llama_cpp": {
      "api_url": "http://localhost:8080"
    }
  }
}
```

В PR также явно добавлена обработка `reasoning_content`, поэтому provider подготовлен к reasoning-моделям, которые отдают reasoning отдельно от обычного assistant text.

## Blame текущей строки можно показывать в status bar

PR [`#59696`](https://github.com/zed-industries/zed/pull/59696) добавляет поле `location` в `git.inline_blame`. Старое поведение остаётся дефолтом: blame показывается inline в конце текущей строки. Новый вариант `status_bar` переносит ту же информацию вниз, в статусную строку.

```json
{
  "git": {
    "inline_blame": {
      "enabled": true,
      "location": "status_bar"
    }
  }
}
```

В реализации это не просто CSS-переключатель. В `crates/editor/src/git.rs` появился `active_git_blame_entry()`, который достаёт `BlameEntry` для строки под курсором, а `crates/git_ui/src/blame_ui.rs` рисует status bar item с Git-иконкой, автором, временем изменения и summary commit'а. Клик по элементу использует прежнюю логику открытия commit details, только точка входа теперь status bar.

Если вам мешает длинный inline blame в строке кода, новый режим сохраняет контекст текущего commit'а, но не сдвигает визуальную ширину строки.

## Breaking notice: `format_on_save` теперь `off` по умолчанию

PR [`#59710`](https://github.com/zed-industries/zed/pull/59710) меняет глобальный дефолт в `assets/settings/default.json`:

```diff
-"format_on_save": "on"
+"format_on_save": "off"
```

Мотивация из PR конкретная: автоматическое форматирование по умолчанию безопасно для экосистем с официальным formatter'ом, но может неожиданно менять файлы там, где форматирование исторически неоднозначно или появилось позже через сторонние инструменты. Поэтому Zed выключает глобальный дефолт, но оставляет `format_on_save: "on"` в language overrides для языков с официальным форматированием.

В diff явно включены, например, `Astro`, `Dart`, `EEx`, `Elixir`, `Elm`, `Go`, `Gleam`, `JSON`, `Jsonnet`, `OCaml`, `Prisma`, `Proto`, `Python`, `Rego`, `Ruby`, `Rust`, `Svelte`, `Terraform`, `TSX`, `TypeScript`, `Vue`, `YAML` и `Zig`. При этом старые явные `"format_on_save": "off"` для `C` и `C++` убраны, потому что теперь это совпадает с глобальным значением.

Если вам нужен прежний режим для всех языков, его теперь лучше зафиксировать явно:

```json
{
  "format_on_save": "on"
}
```

А если нужен точечный режим только для конкретного языка, оставьте глобальный default и переопределите language setting.

## AI-настройки переехали в Settings Editor

PR [`#59860`](https://github.com/zed-industries/zed/pull/59860) убирает старый feature flag для AI settings и переносит ключевые агентские настройки в общий Settings Editor. Release notes перечисляют три группы: LLM providers, external agents и MCP servers.

Это заметное архитектурное изменение UI: большой `crates/agent_ui/src/agent_configuration.rs` и modal `add_llm_provider_modal.rs` удалены, а provider-specific карточки и редакторы переехали ближе к settings infrastructure. Например, provider'ы теперь отдают inline description для Settings Editor; у нового `llama_cpp` это строка `Run local models on your machine with LlamaCpp.`.

Для пользователя результат проще: добавление LLM-провайдера, настройка внешнего агента и подключение MCP-server теперь находятся рядом с остальными настройками Zed, а не в отдельном агентском configuration UI.

## Agent: project rules можно исключить из генерации commit message

PR [`#56867`](https://github.com/zed-industries/zed/pull/56867) добавляет настройку `agent.commit_message_include_project_rules`. До релиза генерация commit message могла включать project rules files — `AGENTS.md`, `CLAUDE.md`, `.rules` и похожие файлы — в prompt вместе с git diff. Это полезно для задач агента, но для commit summary иногда добавляет лишний контекст.

Новый дефолт сохраняет прежнее поведение (`true`), но его можно выключить:

```json
{
  "agent": {
    "commit_message_include_project_rules": false
  }
}
```

Важно, что это отдельное поле рядом с `commit_message_model`, а не вложенная опция model selection: в diff оно добавлено в `AgentSettings`, default settings и settings content, а `git_panel` использует его именно при генерации commit message.

## Очередь сообщений агента: по умолчанию отправка после generation

PR [`#59310`](https://github.com/zed-industries/zed/pull/59310) меняет поведение queued messages в Zed Agent. Раньше идея queue steering была привязана к turn boundary: пользователь мог отправить сообщение так, чтобы оно вмешалось в ближайшую границу хода. Теперь дефолт другой — queued message отправляется в конце текущей generation.

Для сценариев, где всё-таки нужно steering-поведение, в Zed Agent остаётся toggle `Steer`. Внутри PR старый state, завязанный на индексы и синхронизацию в разных частях thread view, заменён отдельным `message_queue` module со stable queue entry ID. Это важно для надёжности UI: запись в очереди больше не должна ломаться от рассинхронизации индексов между view state и thread state.

## Reasoning effort получил уровень `max`

PR [`#59806`](https://github.com/zed-industries/zed/pull/59806) добавляет `ReasoningEffort::Max` и значение `"max"` в список reasoning levels. Это нужно для моделей вроде DeepSeek V4 и GLM 5.2, где `max` не равен прежнему `xhigh`.

Изменение затрагивает OpenAI-compatible и OpenCode providers. В OpenCode нормализация теперь различает `"xhigh"` и `"max"`, а display mapping возвращает отдельный label `Max`. Для OpenAI subscribed models `Max` явно не включён: в diff `ReasoningEffort::Max => return None`, потому что такие модели его не поддерживают.

## Git diff review: solo diff сначала показывает изменённые hunks

В Git-разделе релиза есть два связанных PR про solo diff. [`#59752`](https://github.com/zed-industries/zed/pull/59752) убирает лишние controls: в header остаётся релевантная staging action для текущего состояния файла, а inactive fold toggles не занимают место в split diff. [`#59958`](https://github.com/zed-industries/zed/pull/59958) меняет default view: solo diff теперь открывается не как полный файл, а как changes-only view с excerpted context вокруг изменённых hunks.

В [`#59958`](https://github.com/zed-industries/zed/pull/59958) также добавлен header toggle для переключения между compact changes-only view и full-file view. Это ближе к обычному code review flow: сначала видны только изменения, а полный файл можно открыть, когда нужен широкий контекст.

## Text finder стал удобнее после появления в 1.9.0

В прошлом релизе Zed добавил modal `text finder`; в `v1.10.0` его донастроили двумя маленькими, но практичными изменениями.

- [`#59840`](https://github.com/zed-industries/zed/pull/59840): при открытии результата cursor теперь попадает в точную колонку совпадения. Раньше text finder открывал файл на начале строки, в отличие от project search; теперь используется `SearchMatch::relative_range.start` вместо колонки `0`.
- [`#59849`](https://github.com/zed-industries/zed/pull/59849): text finder запоминает last query и filters. Если вы нашли совпадение, внесли правку и снова открыли finder, query больше не заменяется случайным словом под курсором после редактирования.

Это делает text finder не только навигационным поиском, но и более удобным инструментом для серии быстрых правок по одному search query.

## Markdown preview получил отдельный размер шрифта

PR [`#55489`](https://github.com/zed-industries/zed/pull/55489) добавляет настройку `markdown_preview_font_size` и связанные font family настройки:

```json
{
  "markdown_preview_font_size": 16,
  "markdown_preview_font_family": "Inter",
  "markdown_preview_code_font_family": "JetBrains Mono"
}
```

До изменения shortcuts масштабирования влияли в Markdown preview в основном на code blocks, потому что body использовал UI font size, а headings были завязаны на `rem`. Теперь preview subtree обёрнут в `WithRemSize(buffer_font_size)`: body, headings (`text_3xl`, `text_2xl` и т.д.) и rem-based spacing масштабируются вместе, а внешний focus root и scrollbar остаются на UI font size.

## Project Panel: expand/collapse all стали видимыми в context menu

PR [`#59567`](https://github.com/zed-industries/zed/pull/59567) добавляет action `project_panel::ExpandAllEntries`, shortcut и пункты context menu для `Expand All` / `Collapse All`. На Linux и Windows новый shortcut — `ctrl-right`, на macOS — `cmd-right`; существующий collapse all остаётся на `ctrl-left` / `cmd-left`.

Поведение не глобально-грубое: context menu action применяется к выбранному worktree, а для подпапки `Expand All` раскрывает дерево вниз именно от этой папки.

## Языки и режимы: Go builtins, dynamic semantic tokens и Helix bindings

Несколько изменений полезны тем, кто работает в конкретных language/mode сценариях:

- [`#59780`](https://github.com/zed-industries/zed/pull/59780) добавляет в Go grammar captures `@type.builtin` и `@function.builtin` для universe-scope identifiers. Это выравнивает Go с Rust, TypeScript и Python: темы теперь могут отдельно подсвечивать встроенные типы и функции.
- [`#60015`](https://github.com/zed-industries/zed/pull/60015) включает dynamic registration для `textDocument/semanticTokens`. Это важно для Roslyn/C#: сервер не объявляет `semanticTokensProvider` статически в `initialize`, а регистрирует capability через `client/registerCapability` только если клиент поддерживает dynamic registration.
- [`#59638`](https://github.com/zed-industries/zed/pull/59638) расширяет Helix keymap: `] g` и `[ g` переходят к следующему/предыдущему hunk, `alt-b` / `alt-e` двигаются к началу/концу larger syntax node, `*` ставит текущий selection как search query, а исправления для `[`/`]` убирают зависания single-key follow-ups.

## Что важно проверить после обновления

1. Если вы полагались на автоматическое форматирование во всех языках, проверьте `format_on_save`: глобально он теперь `off`, а прежний режим нужно включить явно.
2. Если inline blame мешал в длинных строках, попробуйте `git.inline_blame.location: "status_bar"`.
3. Если вы запускаете локальные модели через `llama serve`, переведите конфигурацию с generic OpenAI-compatible provider на новый `llama_cpp`, чтобы получить model discovery.
4. Если генерация commit message тащит слишком много policy/context файлов, выключите `agent.commit_message_include_project_rules`.
